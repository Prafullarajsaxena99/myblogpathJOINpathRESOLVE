Using path.join and path.resolve

What Are File Paths

File paths are exactly what they seem like! They provide a readable way to locate and refer to all the files or directories in your PC.
They are of two types:

-Absolute File Paths

Absolute file path provide specifies the complete path of the file, irrespective of where you are at in your PC directories.
For example, if there is a directory ‘Directory_A’ which contains a ‘Directory_B’ which contains a ‘Directory_C’ and some files, and ‘Directory_A’ is contained in your home folder.
Absolute path to a file named ‘file.txt’ contained in ‘Directory C’ will be : /Directory_A/Directory_B/Directory_C/file.txt.

-Relative File Paths

Relative file paths differ from Absolute file path in way that they work with reference to your current directory, where your script is running.

. is used to represent the present working directory.
.. is used to represent the parent directory of the current directory/file.

Taking the same example as Absolute File Paths, if our current working directory would be Directory C, file path for a file file_A.txt contained in Directory A would look like : ../../file_A.txt.

Path modules

When working with Web development using Node.js, learning about what file paths are and how to use them, becomes a key skill.
Path modules provide an simple way to interact with file paths, effectively, efficiently and in an optimized manner.

Node.js has an in-built core module named path that helps us in doing so!

The most commonly used methods from path modules are:

-path.join()
-path.resolve()
-path.dirname()
-path.basename()

I’ll be talking about path.join() and path.resolve() methods in this blog.

Path.resolve()

path.resolve() is used to resolve the file path segments into an absolute path. It keeps adding the path considering them relative, from right to left until it completes creating an absolute path.
If the method finds an absolute path, it ignores the resolved segments, and gives back the absolute path.

Note : If no arguments passed, path.resolve() returns current working directory.

path.resolve() does not work by checking if the file or directory exists or not. It works ob String concatenation.
